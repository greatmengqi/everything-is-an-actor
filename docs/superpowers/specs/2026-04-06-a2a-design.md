# A2A: Agent-to-Agent Multi-Turn Protocol

Single-machine A2A support for `everything-is-an-actor`. Two additions to the existing agent layer. No new communication primitives, no new framework mechanisms.

## Motivation

A2A (Google, 2025) defines how agents discover and converse across boundaries. Two concepts have independent value on single-machine:

1. **Capability discovery** — find agents by what they can do, not by address
2. **Multi-turn** — agents ask each other for information mid-execution

## A2A mapping

| A2A concept | Framework equivalent | New? |
|-------------|---------------------|------|
| Agent Card | `AgentActor.__card__` | Yes |
| Task | `Task[I]` | Existing |
| Task state machine | `TaskStatus` | Existing |
| `input-required` | Actor state + ask loop (usage pattern) | No — actor is already a state machine |
| Message / Part | `Task[I]` type parameter | Existing |
| Artifact | `TaskResult[O].output` | Existing |
| Streaming | `ask_stream()` | Existing |
| Agent Discovery | `AgentSystem.discover(skill)` | Yes |
| Multi-turn | Stateful actor + repeated `ask` | No — just a pattern |

## 1. AgentCard

Static capability metadata on `AgentActor`. Class attribute.

```python
# agents/card.py

@dataclass(frozen=True)
class AgentCard:
    skills: tuple[str, ...] = ()
    description: str = ""
```

```python
class TranslateAgent(AgentActor[str, str]):
    __card__ = AgentCard(
        skills=("translation", "summarization"),
        description="Translates and summarizes documents",
    )
```

Default `AgentCard()` on base `AgentActor` — agents without `__card__` are not discoverable but still functional.

## 2. discover()

Query running actors by skill. No separate registry — iterates `_root_cells` and checks `__card__`.

```python
# On AgentSystem:

def discover(self, skill: str) -> list[ActorRef]:
    """Find spawned agents whose AgentCard declares the given skill."""
    return [
        cell.ref
        for name, cell in self._actor_system._root_cells.items()
        if hasattr(cell.actor_cls, '__card__')
        and skill in cell.actor_cls.__card__.skills
    ]
```

For VirtualActorRegistry: extend with same pattern.

## Multi-turn pattern (no framework change)

Actor is a state machine. Multi-turn is a usage pattern, not a framework feature.

### Child agent — stateful, tracks conversation

```python
class TranslateAgent(AgentActor[str, str]):
    __card__ = AgentCard(skills=("translation",))
    _pending: str | None = None

    async def execute(self, input: str) -> str:
        match self._state:
            case "init":
                self._pending = input
                self._state = "waiting_lang"
                return "Chinese or English?"
            case "waiting_lang":
                self._state = "init"
                return f"{input}: {self._pending}"
```

### Parent agent — ask loop

```python
class OrchestratorAgent(AgentActor[str, str]):
    async def execute(self, input: str) -> str:
        ref = await self.context.spawn(TranslateAgent, "translator")
        result = await self.context.ask(ref, Task(input=input))
        # Convention: if agent needs more input, output is a question string
        # Business logic decides when the conversation is done
        while self.needs_more(result):
            answer = self.decide(result.output)
            result = await self.context.ask(ref, Task(input=answer))
        return result.output
```

### Design rationale

Considered and rejected:
- **Side-channel / InquiryChannel**: extra complexity, not needed
- **callback()**: equivalent to ask, no benefit
- **receive()**: breaks AgentActor abstraction (execute is the API, not mailbox)
- **become()**: too rigid — forces separate methods per state
- **InputRequired / Inquiry types**: framework doesn't need to know about multi-turn — it's business logic
- **converse()**: unnecessary abstraction
- **Message / Sender wrappers**: unnecessary abstraction

Actor is already a state machine. `execute()` is called per message. Self tracks state. Parent decides when the conversation is done. No framework involvement.

## What this does NOT include

- HTTP transport (future `integrations/a2a/`)
- Agent Card versioning or schema negotiation
- Authentication / authorization
- Part ADT (TextPart / FilePart / DataPart)
- Push notifications

## Backward compatibility

- `execute()` unchanged
- `ask()` unchanged
- `tell()` unchanged
- `TaskStatus` unchanged
- Agents without `__card__` work as before
