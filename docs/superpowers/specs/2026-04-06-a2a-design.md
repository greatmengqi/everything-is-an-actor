# A2A: Agent-to-Agent Multi-Turn Protocol

Single-machine A2A support for `everything-is-an-actor`. Three additions to the existing agent layer. No new packages, no new communication primitives beyond exposing what's already in the mailbox.

## Motivation

A2A (Google, 2025) defines how agents discover and converse across boundaries. Two concepts have independent value on single-machine:

1. **Capability discovery** â€” find agents by what they can do, not by address
2. **Multi-turn** â€” agents ask each other for information mid-execution

The current framework lacks both. `ask` is single-shot (input -> output). Agents cannot discover peers by skill. This spec adds both with minimal new surface.

## Design Principles

- No new packages or layers â€” extend `agents/` only
- Multi-turn via `tell` + `receive` â€” symmetric on both sides, pure Actor Model
- No callback, no side-channel, no system polling
- `ask` unchanged â€” single-shot stays single-shot
- `execute()` signature unchanged â€” backward compatible

## 1. AgentCard

Static capability metadata on `AgentActor`. Class attribute (not method â€” metadata doesn't depend on instance state).

```python
# agents/card.py

@dataclass(frozen=True)
class AgentCard:
    skills: tuple[str, ...] = ()
    description: str = ""
```

```python
class TranslateAgent(AgentActor[str, str]):
    __card__ = AgentCard(
        skills=("translation", "summarization"),
        description="Translates and summarizes documents",
    )

    async def execute(self, input: str) -> str:
        ...
```

Default `AgentCard()` on base `AgentActor` â€” agents without `__card__` are not discoverable but still functional.

## 2. Message + receive()

### Message

User-facing projection of `_Envelope`. Wraps body + sender.

```python
# agents/message.py

@dataclass(frozen=True)
class Message(Generic[T]):
    body: T
    sender: Sender | None = None

@dataclass(frozen=True)
class Inquiry:
    """Typed inquiry message for multi-turn conversations."""
    question: str
```

### Sender

Wraps `ActorRef`. Only has `tell` â€” no ask, no callback.

```python
class Sender:
    ref: ActorRef

    async def tell(self, msg: Any) -> None:
        """Send a message to this sender. Fire-and-forget."""
        ...
```

### receive()

Exposes `mailbox.dequeue()` to the actor within `execute()`. Erlang-style.

```python
# On ActorContext:

async def receive(self, *, timeout: float = 30.0) -> Message:
    """Pull the next message from this actor's mailbox.

    Blocks until a message arrives or timeout expires.
    Raises TimeoutError on timeout.
    """
```

Implementation: calls `self._cell.mailbox.dequeue()` directly, wraps the envelope as `Message`. Safe because the run loop is suspended at `await on_receive()` â€” only one consumer at a time.

Stop sentinels: if `receive()` dequeues a `_Stop`, re-enqueue it and raise `ActorStoppedError`.

### Sender propagation

`tell()` and `ask()` populate `envelope.sender` with the caller's `self_ref`:

- `context.tell(ref, msg)` â†’ `_Envelope(payload=msg, sender=self.context.self_ref)`
- `context.ask(ref, msg)` â†’ `_Envelope(payload=msg, sender=self.context.self_ref, correlation_id=...)`
- `system.tell(ref, msg)` â†’ `_Envelope(payload=msg, sender=None)` (external caller)
- `system.ask(ref, msg)` â†’ `_Envelope(payload=msg, sender=None, correlation_id=...)`

Framework sets `self.message` on the actor before calling `execute()`:

```python
# In _ActorCell._run(), before on_receive:
actor.message = Message(
    body=envelope.payload,
    sender=Sender(ref=envelope.sender) if envelope.sender else None,
)
```

### Multi-turn example

Both sides use `tell` + `receive`. Symmetric.

```python
# Agent B (child â€” needs more info)
class TranslateAgent(AgentActor[str, str]):
    __card__ = AgentCard(skills=("translation",))

    async def execute(self, input: str) -> str:
        # Ask parent for language preference
        await self.message.sender.tell(Inquiry("Chinese or English?"))
        # Wait for answer
        msg = await self.context.receive()
        lang = msg.body
        result = f"{lang}: {input}"
        # Send result back to parent
        await self.message.sender.tell(result)
        return result
```

```python
# Agent A (parent â€” delegates and handles inquiries)
class OrchestratorAgent(AgentActor[str, str]):
    async def execute(self, input: str) -> str:
        ref = await self.context.spawn(TranslateAgent, "translator")
        # tell â€” non-blocking, mailbox stays free
        await self.context.tell(ref, Task(input=input))
        # Process messages until final result
        while True:
            msg = await self.context.receive()
            match msg.body:
                case Inquiry(question=q):
                    await msg.sender.tell("Chinese")
                case str() as result:
                    return result
```

### Single-shot vs multi-turn

| | Single-shot | Multi-turn |
|--|-------------|------------|
| Parent calls | `ask(ref, task)` | `tell(ref, task)` + `receive()` loop |
| Child returns | `return result` (resolves promise) | `sender.tell(result)` + `return` |
| Child inquires | N/A | `sender.tell(Inquiry(...))` + `receive()` |
| Parent handles inquiry | N/A | `receive()` â†’ match â†’ `sender.tell(answer)` |

Single-shot uses `ask` â€” unchanged. Multi-turn uses `tell` + `receive` â€” both sides symmetric.

## 3. discover()

Query running actors by skill. No separate registry â€” iterates `_root_cells` and checks `__card__`.

```python
# On AgentSystem:

def discover(self, skill: str) -> list[ActorRef]:
    """Find spawned agents whose AgentCard declares the given skill."""
    return [
        cell.ref
        for name, cell in self._actor_system._root_cells.items()
        if hasattr(cell.actor_cls, '__card__')
        and skill in cell.actor_cls.__card__.skills
    ]
```

For VirtualActorRegistry: extend with same pattern â€” check `actor_cls.__card__` on registered types.

## External caller API: converse()

For non-actor callers (external code), `AgentSystem` provides `converse()`:

```python
async def converse(
    self,
    agent_cls: type[AgentActor],
    input: Any,
    *,
    timeout: float = 300.0,
) -> AsyncIterator[Message]:
    """Multi-turn interaction for external callers.

    Internally spawns a relay actor that:
    1. Spawns the target agent
    2. Tells the task (with relay as sender)
    3. Receives messages and yields them to the caller
    4. Forwards caller's replies back to the target
    """
```

```python
# External usage
async for msg in system.converse(TranslateAgent, "hello world"):
    match msg.body:
        case Inquiry(q):
            await msg.sender.tell("Chinese")
        case result:
            print(result)
            break
```

## What this does NOT include

- HTTP transport (future `integrations/a2a/`)
- Agent Card versioning or schema negotiation
- Authentication / authorization
- Part ADT (TextPart / FilePart / DataPart)
- Push notifications

## Backward compatibility

- `execute()` signature unchanged: `async def execute(self, input: I) -> O`
- `ask()` behavior unchanged: single-shot, promise-based
- `tell()` behavior: sender now populated, otherwise unchanged
- `on_receive()` unchanged
- Agents without `__card__` work as before
- `self.message` is a new attribute â€” no conflict with existing code
- `receive()` is opt-in â€” agents that don't call it are unaffected
