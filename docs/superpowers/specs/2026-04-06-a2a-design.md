# A2A: Agent-to-Agent Multi-Turn Protocol

Single-machine A2A support for `everything-is-an-actor`. Three additions to the existing agent layer. No new communication primitives.

## Motivation

A2A (Google, 2025) defines how agents discover and converse across boundaries. Two concepts have independent value on single-machine:

1. **Capability discovery** â€” find agents by what they can do, not by address
2. **Multi-turn** â€” agents ask each other for information mid-execution

## A2A mapping

| A2A concept | Framework equivalent | New? |
|-------------|---------------------|------|
| Agent Card | `AgentActor.__card__` | Yes |
| Task | `Task[I]` | Existing |
| Task state machine | `TaskStatus` | Existing |
| `input-required` | `TaskStatus.INPUT_REQUIRED` + `Inquiry` | New enum value + type |
| Message / Part | `Task[I]` type parameter | Existing |
| Artifact | `TaskResult[O].output` | Existing |
| Streaming | `ask_stream()` | Existing |
| Agent Discovery | `AgentSystem.discover(skill)` | Yes |
| Multi-turn | `execute()` returns `Inquiry` â†’ parent `ask`s again | Convention, no new mechanism |

## Design Principles

- No new packages or layers â€” extend `agents/` only
- No new communication primitives â€” multi-turn is just `ask` in a loop
- `execute()` signature unchanged â€” returns `Inquiry` for "not done yet", normal value for "done"
- Backward compatible â€” existing agents unaffected

## 1. AgentCard

Static capability metadata on `AgentActor`. Class attribute.

```python
# agents/card.py

@dataclass(frozen=True)
class AgentCard:
    skills: tuple[str, ...] = ()
    description: str = ""
```

```python
class TranslateAgent(AgentActor[str, str]):
    __card__ = AgentCard(
        skills=("translation", "summarization"),
        description="Translates and summarizes documents",
    )
```

Default `AgentCard()` on base `AgentActor` â€” agents without `__card__` are not discoverable but still functional.

## 2. Inquiry + INPUT_REQUIRED

### Inquiry type

```python
# agents/message.py

@dataclass(frozen=True)
class Inquiry:
    """Returned from execute() to signal the agent needs more input."""
    question: str
```

### TaskStatus extension

```python
class TaskStatus(Enum):
    PENDING = "pending"
    RUNNING = "running"
    COMPLETED = "completed"
    FAILED = "failed"
    CANCELLED = "cancelled"
    INPUT_REQUIRED = "input_required"  # new
```

Framework behavior: if `execute()` returns an `Inquiry`, the framework wraps it in `TaskResult` with `status=INPUT_REQUIRED`. Otherwise `status=COMPLETED` as before.

### Multi-turn example

Child agent â€” returns `Inquiry` when it needs more info:

```python
class TranslateAgent(AgentActor[str, str]):
    __card__ = AgentCard(skills=("translation",))
    _pending: str | None = None

    async def execute(self, input: str) -> str | Inquiry:
        if self._pending is None:
            self._pending = input
            return Inquiry("Chinese or English?")
        else:
            result = f"{input}: {self._pending}"
            self._pending = None
            return result
```

Parent agent â€” asks in a loop until done:

```python
class OrchestratorAgent(AgentActor[str, str]):
    async def execute(self, input: str) -> str:
        ref = await self.context.spawn(TranslateAgent, "translator")
        result = await self.context.ask(ref, Task(input=input))
        while result.status == TaskStatus.INPUT_REQUIRED:
            answer = self.decide(result.output.question)
            result = await self.context.ask(ref, Task(input=answer))
        return result.output
```

Each round is a normal `ask` â†’ `TaskResult`. No new communication mechanism.

### Multiple rounds

The child can return `Inquiry` as many times as needed. Each round is a separate `execute()` call. The child maintains state via `self`:

```python
class DetailedTranslateAgent(AgentActor[str, str]):
    __card__ = AgentCard(skills=("translation",))
    _text: str | None = None
    _lang: str | None = None

    async def execute(self, input: str) -> str | Inquiry:
        if self._text is None:
            self._text = input
            return Inquiry("Chinese or English?")
        elif self._lang is None:
            self._lang = input
            return Inquiry("Formal or casual?")
        else:
            result = f"{input} {self._lang}: {self._text}"
            self._text = self._lang = None
            return result
```

## 3. discover()

Query running actors by skill. No separate registry â€” iterates `_root_cells` and checks `__card__`.

```python
# On AgentSystem:

def discover(self, skill: str) -> list[ActorRef]:
    """Find spawned agents whose AgentCard declares the given skill."""
    return [
        cell.ref
        for name, cell in self._actor_system._root_cells.items()
        if hasattr(cell.actor_cls, '__card__')
        and skill in cell.actor_cls.__card__.skills
    ]
```

For VirtualActorRegistry: extend with same pattern.

## What this does NOT include

- HTTP transport (future `integrations/a2a/`)
- Agent Card versioning or schema negotiation
- Authentication / authorization
- Part ADT (TextPart / FilePart / DataPart)
- Push notifications
- Message / Sender / receive() â€” not needed

## Backward compatibility

- `execute()` signature unchanged â€” `Inquiry` return is opt-in
- `ask()` behavior unchanged
- `tell()` unchanged
- `TaskStatus` gains one value â€” exhaustive `match` on TaskStatus needs updating
- Agents without `__card__` work as before
